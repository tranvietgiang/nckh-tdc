# Quick Reference - Authentication System\n\n## Environment Variables Needed\n\n### Backend (.env)\n`\nAPP_NAME=NCKH-TDC\nAPP_ENV=production\nAPP_DEBUG=false\nAPP_KEY=base64:...\n\nDB_CONNECTION=mysql\nDB_HOST=127.0.0.1\nDB_PORT=3306\nDB_DATABASE=nckh_tdc\nDB_USERNAME=root\nDB_PASSWORD=\n\nSANCTUM_STATEFUL_DOMAINS=localhost:3000,127.0.0.1:3000\nSANCTUM_TOKEN_PREFIX=1\n\nRATE_LIMITER=redis  # or database\nREDIS_HOST=127.0.0.1\nREDIS_PORT=6379\n`\n\n### Frontend (.env.local)\n`\nVITE_API_URL=http://localhost:8000/api\n`\n\n---\n\n## Common Commands\n\n### Backend Commands\n`bash\n# Clear rate limiting\nphp artisan tinker\n>>> Illuminate\\Support\\Facades\\RateLimiter::clear('login.lockout:username')\n>>> exit\n\n# Check failed logins\nphp artisan tinker\n>>> App\\Models\\ActivityLog::where('action', 'login')->latest()->limit(10)->get()\n\n# Clear all auth tokens\nphp artisan tinker\n>>> DB::table('personal_access_tokens')->truncate()\n`\n\n### Frontend Development\n`bash\ncd frontend-reactjs\nnpm install\nnpm run dev\n`\n\n---\n\n## HTTP Status Codes in Login\n\n| Code | Scenario | Response |\\n|------|----------|----------|\\n| 200 | Success | `{success: true, user: {...}, token: \"...\"}` |\\n| 401 | Wrong credentials | `{success: false, message: \"Sai tài khoản hoặc mật khẩu!\"}` |\\n| 422 | Role mismatch | `{success: false, message: \"Tài khoản này không phải ...!\"}` |\\n| 429 | Account locked | `{success: false, message: \"Tài khoản bị khóa tạm thời...\"}` |\\n\n---\n\n## Role Mapping\n\n`\nFrontend       Backend Database\nstudent    →   student\nlecturer   →   teacher\nadmin      →   admin\n`\n\n---\n\n## Token Header Format\n\n`\nAuthorization: Bearer 1|AbCdEfGhIjKlMnOpQrStUvWxYz...\n`\n\n---\n\n## Debug Tips\n\n### Check if token is stored\n`javascript\nlocalStorage.getItem('auth_token')\nlocalStorage.getItem('auth_user')\n`\n\n### Monitor API calls\nF12 → Network → Filter by XHR/Fetch\n\n### Check user role in console\n`javascript\nconst user = JSON.parse(localStorage.getItem('auth_user'))\nconsole.log(user.role)\n`\n\n### Verify token validity\n`php\n// In tinker\n$token = 'your_token';\n$model = \\Laravel\\Sanctum\\PersonalAccessToken::findToken($token);\n$model->tokenable; // returns user\n`\n\n---\n\n## Troubleshooting Matrix\n\n| Problem | Check |\\n|---------|-------|\\n| Login fails with 401 | Username/password correct, user exists in DB |\\n| Login fails with 422 | User role matches selected role |\\n| Login fails with 429 | Wait 15 mins or clear RateLimiter |\\n| Token not persisting | Check localStorage, verify AuthProvider |\\n| Routes return 401 | Token format correct, has Bearer prefix |\\n| Routes return 403 | User role matches required role |\\n| CORS errors | Check config/cors.php, frontend URL allowed |\\n| Activity not logging | Check activity_logs table, verify creation |\\n\n---\n\n## Testing Single Feature\n\n### Test role validation only\n`bash\n# 1. Start Laravel\ncd backend-laravel\nphp artisan serve\n\n# 2. In another terminal, test API\ncurl -X POST http://localhost:8000/api/login \\\\\\n  -H \"Content-Type: application/json\" \\\\\\n  -d '{\"username\":\"student1\",\"password\":\"password\",\"user_role\":\"lecturer\"}'\n\n# Expected: 422 error\n`\n\n### Test lockout\n`bash\n# Run 6 times with wrong password\nfor i in {1..6}; do\n  curl -X POST http://localhost:8000/api/login \\\\\\n    -H \"Content-Type: application/json\" \\\\\\n    -d '{\"username\":\"student1\",\"password\":\"wrong\",\"user_role\":\"student\"}'\n  echo \"\\nAttempt $i\"\ndone\n\n# 6th attempt should return 429\n`\n\n---\n\n## Performance Benchmarks\n\n- Login request: ~100-200ms (with DB queries)\n- Protected route check: ~5-10ms\n- Token validation: ~2-5ms\n- Rate limit check: ~1-3ms\n\n---\n\n## Security Checklist Before Production\n\n- [ ] HTTPS enforced\n- [ ] CORS properly configured (not wildcard)\n- [ ] Rate limiting enabled\n- [ ] Activity logging working\n- [ ] Passwords hashed with bcrypt\n- [ ] Token expiration set\n- [ ] Admin credentials strong\n- [ ] Database backups scheduled\n- [ ] Logs monitored\n- [ ] Error handling doesn't leak info\n
